API说明

接口格式
－ 请求： https://wx.firstwisdom.cn/  
－ 返回： json格式
         {"data":"","info":"","status":0/1/2}
         status：为0表示成功，status不为0则表示失败, 其中2表示未登录。
         info：提示信息或错误信息
         data：成功时返回的数据，json格式。
－ 请求方法： GET POST 不限

========== app和服务端session机制 ==========
- app第一次调用后端接口时,接口返回一个名字为 GEKSESSID 的cookie,存放的是session id。
  app接收到这个session id之后需要保存到本地,并且之后每次调用接口,都需要通过cookie带上这个session id。

- app每次调用后端接口,都要检查接口是否返回一个名字为 GEKSESSID 的cookie,如果返回,则需要更新之前保存的接口session id。
  因为后端对于已登录的session id,会定期重新生成, 加强安全性。

- 对于需要登录的接口,如果用户未登录,则接口返回 {"data":"","info":"","status":2} status等于2表示未登录,需要跳转到登录界面,让用户登录。

- app不需要考虑GEKSESSID cookie的有效期和作用域,session id是否有效由后端决定。

========== 订阅/提醒接口 ==========
- 检查用户是否订阅
  https://wx.firstwisdom.cn/stocks/isSubscribe?symbol=000001&type=1
  参数: 
     symbol: 股票代码
     type: 服务类型: 1- T+0提醒 2-多空提醒 
  返回data:  0-未订阅 1-已订阅

- 列出用户订阅服务
  https://wx.firstwisdom.cn/stocks/listsubscribe?symbol=000001
  参数:
    symbol: 股票代码, 【可选】,如果没有指定股票代码,则返回用户自选股票的订阅服务。
  返回data: 
    数组,格式为: data[symbol][type] = status
      symbol: 股票代码
      type: 服务类型: 1- T+0提醒 2-多空提醒 
      status: 0-未订阅 1-已订阅
    
  注意: 用户自选股票没有出现在返回结果中时,则视为未订阅该服务。

- 订阅
  https://wx.firstwisdom.cn/stocks/subscribe?symbol=000001&type=1
  参数: 
     symbol: 股票代码
     type: 服务类型: 1- T+0提醒 2-多空提醒

- 取消订阅
  https://wx.firstwisdom.cn/stocks/cancelsubscribe?symbol=000001&type=2
  参数: 
     symbol: 股票代码
     type: 服务类型: 1- T+0提醒 2-多空提醒

  https://wx.firstwisdom.cn/order/getUserServices 
  返回data说明:
  id: 服务id
  status: 0-未过期  1- 已过期
  start_date: 服务开始时间
  end_date: 服务结束时间

- 获取服务的信息
  https://wx.firstwisdom.cn/order/getService?service_id=1
  参数说明：
    service_id : 1=一智金股，2=涨停大师

  id: 服务id
  name: 服务名
  icon: 服务图标URL
  intro: 服务介绍
  sku: 服务sku

     type: sku 类型  1-一个月 3-三个月 12-一年
     price: 价格
     mark_price: 原价

- 提交订单并支付
  https://wx.firstwisdom.cn/order/buy?sku_id=1&fr=1&pay_type=2
  参数：
    sku_id: 商品ID    一智金股： 6 涨停大师：3

  返回data说明:
   order_sn: 订单编号
   order_id: 订单ID
   pay: 支付的请求参数

- #获取授权链接
    https://wx.firstwisdom.cn/Misc/r

    参数：
    redirect ： 指定获取头像之后跳转的链接(例如："https://wx.firstwisdom.cn/#/user", 就填 "/#/user")
    state : get_info(登录注册流程获取头像信息)/do_login(尝试使用微信登陆)

    返回数据：
    {"data":"https:\/\/open.weixin.qq.com\/connect\/oauth2\/authorize?appid=wxff3ee0fea37b509b&redirect_uri=http%3A%2F%2Fapi.firstwisdom.cn%2F&response_type=code&scope=snsapi_userinfo&state=head_img#wechat_redirect","info":"success","status":0}

    接口解析：
    流程分2种情况：
     1、该用户未设置过头像
     --> 用户注册/登录
     --> 【前端】判断用户头像是否存在
     -->  不存在
     --> 【前端】请求"获取授权链接"接口(即本接口)
     --> 【前端】(location.href)跳转接口返回data中的url
     --> 【后台】获取头像，跳转至前端指定的 redirect
     --> 【前端】在redirect页面，通过https://wx.firstwisdom.cn/user/get重新获取用户信息
     -->  展示头像

     2、该用户设置过头像
     --> 用户注册/登录
     --> 【前端】判断用户头像是否存在
     -->  存在
     -->  展示头像
